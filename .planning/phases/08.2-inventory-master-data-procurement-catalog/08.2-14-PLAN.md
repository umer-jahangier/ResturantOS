`category-tree.tsx` exports `CategoryTree({ nodes, onEdit, onAddChild, onMove, onArchive, onRestore,
selectedId })` and an internal recursive `CategoryTreeRow`. No tree primitive exists in this
codebase, so assemble from `Button variant="ghost"` rows plus lucide `ChevronRight` / `ChevronDown`
for the expand toggle and `DropdownMenu` for row actions. Indent by depth with a left-padding step;
do not use a nested list element with default list styling.

Each row shows: the expand toggle (only when the node has children), the name, the ingredient count,
and the three default GL account codes as muted chips, each carrying an "(inherited)" suffix in
`text-muted-foreground` when the corresponding inherited flag on `resolvedGlAccounts` is true. An
archived node renders the `archived` StatusBadge variant and its action menu offers Restore instead
of Archive.

Row action menu entries: Edit, Add subcategory, Move to, Archive. The Add-subcategory entry is
omitted entirely when the node's level is 3 — hidden, not disabled with a tooltip, per the UI
contract.

The Move-to entry opens a nested menu listing only valid targets: exclude the node itself, exclude
every descendant of the node (walk the subtree client-side), exclude any node already at level 3,
and exclude any node where the new parent's level plus the moved subtree's own depth would exceed 3.
Also offer a "Make top-level" entry. Selecting a target calls `onMove(nodeId, targetIdOrNull)`.

Accessibility: the container gets `role="tree"`, rows get `role="treeitem"` with `aria-level` and,
where applicable, `aria-expanded`. Expand toggles are real buttons with an accessible name. Icon-only
action buttons carry the existing `.touch-target` utility for the 44px minimum.

`CategoryFormDialog.tsx` exports `CategoryFormDialog({ category, defaultParentId, trigger })`.
Fields: name (required), code, parent (a raw styled select listing only categories at level below 3,
disabled in edit mode because reparenting is the Move flow), the three default GL account code
inputs, variance cap percentage, an exclude-from-purchase-order-suggestions toggle button (no
checkbox primitive exists — use the `aria-pressed` button pattern from `allergen-pill-toggle`), and
sort order. Validation is on blur for text fields and on submit for the form, via
`createZodResolver`. Use the Copywriting Contract's strings verbatim for the dialog title,
description and submit button.

Typography: page and section titles at weight 600, body at 400. `font-medium` only via `FormLabel`
or on a sub-heading directly inside a dynamic line-list section.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser user → category mutations | Write actions must be gated by permission before rendering, and again server-side. |
| Server 409 payload → inline error text | Untrusted server message rendered into the dialog. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08.2-141 | Elevation of Privilege | Category write controls visible to a view-only role | medium | mitigate | Every mutating control is wrapped in `PermissionGuard require="inventory.item.manage"`; the server authorizes independently, so the guard is defence in depth, not the control. |
| T-08.2-142 | Repudiation | An archive refusal disappearing as a toast, leaving the user believing it worked | high | mitigate | The refusal renders inline in the still-open dialog with `role="alert"`; `archiveRefusalKeepsTheDialogOpenAndShowsTheServerMessage` gates it. |
| T-08.2-143 | Tampering | A client-side move that would exceed the depth cap or create a cycle | medium | mitigate | The move-target list excludes self, descendants and level-3 nodes client-side, and the service plus the database trigger reject it independently. |
| T-08.2-144 | Spoofing | Server-supplied error text rendered as markup | low | mitigate | The message is rendered as a React text child; no `dangerouslySetInnerHTML` is used. |
</threat_model>
