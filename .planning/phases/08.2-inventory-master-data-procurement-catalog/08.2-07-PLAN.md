<objective>
Add the non-persisting recipe cost-preview endpoint the live plate-cost panel needs. The UI contract
makes this a deliberate server round-trip rather than browser arithmetic, because cross-dimension
UOM conversion and moving-average costing are backend-owned invariants and duplicating them in the
browser risks silently drifting from the production depletion math.

Purpose: ROADMAP Success Criterion 3's "live plate-cost panel showing batch cost, cost per portion,
food-cost % and each line's share of plate cost".
Output: `POST /api/v1/inventory/recipes/preview`, a pure `RecipeCostPreviewService`, and both a unit
test and an integration test.
</objective>

- `RecipeCostPreviewDto(long batchCostPaisa, long portionCostPaisa, BigDecimal yieldServings, Long menuItemPricePaisa, BigDecimal foodCostPct, int excludedLineCount, List<RecipeCostLineDto> lines)` — `foodCostPct` is null when `menuItemPricePaisa` is null or zero.

`RecipeCostPreviewService`, `@Service`, class-level `@Transactional(readOnly = true)`,
constructor-injecting `IngredientRepository`, `IngredientBranchStockRepository`,
`UnitOfMeasureRepository`, `MenuItemCatalogRepository`, `TenantContext`.

`preview(PreviewRecipeCostRequest request)`:
1. Resolve `tenantId` via `tenantContext.requireTenantId()`.
2. Batch-load every distinct `ingredientId` in the request in ONE `findAllById` and every referenced
   UOM code in ONE query; build lookup maps. No per-line repository calls.
3. Batch-load the `IngredientBranchStock` rows for `(tenantId, request.branchId, ingredientIds)` in
   one query and build a `Map<UUID, Long>` of `avgCostPaisa`.
4. For each line, in order: construct an unsaved `RecipeLine` exactly as `RecipeService.createVersion`
   does (ingredientId, qty, uomCode, yieldPct defaulting to 100), then compute
   `effectiveBaseQty = UomConverter.effectiveBaseQty(line, 1, request.yieldServings(), toBaseFactor)`
   where `toBaseFactor` comes from the UOM lookup. Line cost in paisa is `effectiveBaseQty` multiplied
   by the ingredient's `avgCostPaisa` as BigDecimal, rounded HALF_UP to zero decimals exactly once.
   Emit a warning and skip the line (no cost, excluded from the total) when: the ingredient id is
   unknown; the uom code is unknown; the resolved UOM's base differs from the ingredient's
   `baseUomCode` so no in-dimension conversion exists; or no stock row / a zero `avgCostPaisa` means
   the ingredient has never been costed. Warning text must be the UI contract's copy, interpolating
   the ingredient name.
5. `batchCostPaisa` is the sum of the successfully costed lines. `portionCostPaisa` is
   `batchCostPaisa` divided by `yieldServings`, BigDecimal, HALF_UP to zero decimals.
   `sharePctOfBatch` is the line's cost over `batchCostPaisa` times 100, HALF_UP to a whole number;
   it is 0 for every line when `batchCostPaisa` is zero (never divide by zero).
6. When `menuItemId` is supplied, look up the active `menu_item_catalog` row for the tenant and set
   `menuItemPricePaisa`; `foodCostPct` is `portionCostPaisa / menuItemPricePaisa * 100` as a
   BigDecimal at scale 1, HALF_UP, and null when the price is null or zero.

Every intermediate is BigDecimal. No `double`, no `float`, no chained rounding, and no persistence
call of any kind.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → `POST /api/v1/inventory/recipes/preview` | Untrusted draft line array, `branchId` and `menuItemId`. |
| Preview service → ingredient_branch_stock | Reads per-branch moving-average cost, commercially sensitive. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08.2-071 | Information Disclosure | Per-ingredient cost exposed to read-only roles | high | mitigate | The endpoint authorizes at `authorizeManage` level, not `authorizeView`; `viewOnlyPrincipalCannotPreview` gates it. |
| T-08.2-072 | Information Disclosure | Caller-supplied `branchId` reading a sibling branch's costs | high | mitigate | Controller rejects any `branchId` not equal to `claims.branchId()`; `previewRejectsAForeignBranchId` gates it. |
| T-08.2-073 | Tampering | A "preview" that silently writes recipe rows | high | mitigate | The service contains no persistence call (grep-gated) and `previewDoesNotPersistAnything` asserts row counts are unchanged. |
| T-08.2-074 | Denial of Service | A very large draft line array causing an unbounded query fan-out | medium | mitigate | All lookups are batched into a fixed number of queries independent of line count; Bean Validation bounds the payload through `@NotEmpty @Valid`. |
| T-08.2-075 | Repudiation | Authoring estimate diverging from ledger cost | medium | accept | The panel is explicitly labelled an estimate in the UI caption; actual cost is set at depletion time from the same `avgCostPaisa` field, so drift is bounded to the time between preview and sale. |
</threat_model>
