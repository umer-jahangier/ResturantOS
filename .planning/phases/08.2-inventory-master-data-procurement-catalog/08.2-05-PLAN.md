carried-over-defects list owes two guards for the coverage bug, plan 08.2-03 delivers the backend

1. `returns local midnight, not UTC midnight` — call with `"2026-07-22"`, feed the result into
   `new Date(...)`, and assert `getFullYear()` is 2026, `getMonth()` is 6, `getDate()` is 22 and
   `getHours()`, `getMinutes()`, `getSeconds()`, `getMilliseconds()` are all 0. These are LOCAL
   accessors, so the assertion says "local midnight" in a way that holds in every zone the suite
   might run in.
2. `differs from UTC midnight by exactly the local offset` — assert
   `Date.parse(result) - Date.parse("2026-07-22T00:00:00.000Z")` equals
   `new Date(2026, 6, 22).getTimezoneOffset() * 60_000`. This is the inversion guard stated

   positive-offset zone and the test fails. Zone-independent, so it is also a real assertion under
