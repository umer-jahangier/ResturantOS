integration tests.

  <action>
Create `StockLevelDtos` in the `dto` package following `InventoryDtos`' exact container shape (final
class, private constructor, nested records, javadoc naming the requirement).

`StockLevelDto` fields: `UUID ingredientId`, `String ingredientName`, `String sku`,
`String baseUomCode`, `UUID categoryId`, `String categoryName`, `BigDecimal qtyOnHand`,
`BigDecimal reorderPoint`, `long avgCostPaisa`, `long stockValuePaisa`, `Instant lastCountedAt`,
`boolean belowReorderPoint`, `boolean nonPositive`.

`categoryId`/`categoryName` are nullable in this plan and are populated by plan 08.2-09 once the
ingredient DTO exposes them; declare the fields now so the frontend schema does not have to change
twice. Do not add a dependency on `item_categories` here.

`stockValuePaisa` is `qtyOnHand` multiplied by `avgCostPaisa`, rounded HALF_UP to a whole paisa —
compute it once in the service using BigDecimal, never in the browser (D-08: round only at the
display/GL boundary and never chain conversions).

`belowReorderPoint` is `qtyOnHand.compareTo(reorderPoint) <= 0 && reorderPoint.signum() > 0`.
`nonPositive` is `qtyOnHand.signum() <= 0`. These two flags exist so the UI's warning
(`bg-warning/10`) and destructive (`bg-destructive/10`) row washes are server-derived and cannot
drift from backend semantics the way the coverage bug did.

`StockLevelsResponse(UUID branchId, List<StockLevelDto> items, BigDecimal totalStockValuePaisa)` —
actually declare `totalStockValuePaisa` as `long`.

On `IngredientBranchStockRepository`, add
`List<IngredientBranchStock> findByTenantIdAndBranchIdOrderByIngredientIdAsc(UUID tenantId, UUID branchId)`.
Add an explicit `tenantId` predicate rather than relying on RLS + the Hibernate tenant filter alone —
this is the same defect CONTEXT.md flags on `RecipeRepository.findByMenuItemIdOrderByVersionDesc`,
and new code must not repeat it.

  <action>
Create `StockLevelService` as a `@Service` with class-level `@Transactional(readOnly = true)`,
constructor-injecting `IngredientBranchStockRepository`, `IngredientRepository` and `TenantContext`.

`listStockLevels(UUID branchId, String search)` resolves `tenantId` via
`tenantContext.requireTenantId()`, loads the branch's stock rows with the new repository method, then
loads every referenced `Ingredient` in ONE call (`ingredientRepository.findAllById(distinctIds)`)
and builds a `Map<UUID, Ingredient>` before the mapping loop. Do not call `findById` inside the loop —
that is the exact N+1 shape plan 08.2-06 is fixing elsewhere in this service.

Include ingredients that have no `ingredient_branch_stock` row for this branch yet, rendered with
`qtyOnHand = BigDecimal.ZERO`, `avgCostPaisa = 0`, `lastCountedAt = null` — the screen must show an
ingredient a manager has created but never received, otherwise it looks like the ingredient
vanished. Archived/inactive ingredients with zero on-hand are excluded; archived ingredients that
still hold stock are included so the stock is not silently invisible.

`search` is an optional case-insensitive substring filter applied to ingredient name and sku in the
service layer (the UI contract's hard gate is substring matching; fuzzy matching is an explicit
stretch goal, not required).

Sort the result by ingredient name ascending, case-insensitive.

Create `StockLevelController` as a `@RestController` mapped to `/api/v1/inventory/stock`, annotated
`@RequiresFeature("FEATURE_INVENTORY")`, with a single `@GetMapping` taking
`@RequestParam UUID branchId`, `@RequestParam(required = false) String search` and
`@AuthenticationPrincipal JwtClaims claims`. Call `authz.authorizeView(claims.tenantId(),
claims.branchId())` as the first statement, then — before delegating — reject a `branchId` query
parameter that does not equal `claims.branchId()` with an `AccessDeniedException`, so a manager
cannot read a sibling branch's stock by editing the query string. Return
`ResponseEntity.ok(ApiResponse.ok(stockLevelService.listStockLevels(branchId, search)))`.

Add no other HTTP verbs on this controller.
