  <action>
Change `lineFormSchema`: replace the ingredient-identifier field with `vendorItemId: z.string().min(1,
...)` whose message is the field error to show when nothing has been picked. Make `uom` optional (the
server fills it from the catalog entry) while keeping its existing validation when a value is
present, and keep the quantity and unit-price rules as they are. Delete the old field's UUID
validation message entirely — it is the artefact Success Criterion 6 names.

Update `EMPTY_LINE` and `defaultValues()` accordingly.

Watch the header's vendor field with `form.watch("vendorId")` and pass it to `useVendorItems`. Map the
returned catalog rows to `CatalogItemOption`: `id` is the vendor item id, `name` is the linked
ingredient's display name (fall back to the vendor description when the ingredient name is not on the
payload), `secondary` is the pack description with pack quantity and order unit, `sku` is the vendor
SKU, and `pricePaisa` is the current price.

Replace the line row's first cell with `CatalogItemCombobox`. When no vendor is selected, pass
`disabled` and the UI contract's "Select a vendor first." placeholder. Pass the Copywriting
Contract's purchase-order-picker empty heading and body, interpolating the query into the heading.

On selection, set three form values at once: the line's `vendorItemId`, its `uom` from the catalog
row's order unit, and its unit price in rupees converted from the catalog row's current price in
paisa. All three stay editable afterwards so a manager can still book a one-off deal — only the item
itself is picker-only.

When the header vendor changes after lines have been entered, clear every line's `vendorItemId`,
`uom` and price and show a `sonner` info toast explaining that lines were cleared because the vendor
changed. Leaving stale lines pointing at another vendor's catalog would be rejected by the server
with a 422 the user cannot interpret.

Adjust `toPurchaseOrderInput` to emit `vendorItemId` per line and to omit `uom` and the unit price
when they are unchanged from the catalog values, letting the server derive them. Adjust the line-row
grid's column template so the picker cell has enough width to show name, pack and SKU without
truncation at the dialog's current size.

Do not convert this dialog to a routed page and do not create a purchasing-local copy of the
combobox.
