`ItemCategoryService`, `@Service`, class-level `@Transactional(readOnly = true)`, constructor-injecting
`ItemCategoryRepository`, `IngredientRepository`, `TenantContext`.

Reads:
- `listCategories(boolean includeArchived)` — flat list, tenant-scoped, sorted by sortOrder then name,
  each DTO carrying its `ingredientCount` and `resolvedGlAccounts`. Compute ingredient counts with a
  single grouped query or a single `findAll` + in-memory grouping — never one count query per node.
- `getTree(boolean includeArchived)` — builds `ItemCategoryNodeDto` roots from the same single fetch;
  assemble in memory from a `Map<UUID, List<ItemCategory>>` keyed by parentId. One repository call.
- `getCategory(UUID id)`.
- `resolveGlAccounts(ItemCategory node, Map<UUID, ItemCategory> byId)` — private; for each of the
  three account roles, return the node's own code when non-blank (inherited = false), else walk
  `parentId` upward returning the first non-blank ancestor code (inherited = true), else null.

Writes (each `@Transactional`):
- `create(CreateItemCategoryRequest)` — resolve tenant; if `parentId` is null set `level = 1`,
  otherwise load the parent under the same tenant, reject with an `IllegalArgumentException`-family
  domain exception when the parent's level is already 3, and set `level = parent.level + 1`. Always
  set `tenantId` from `tenantContext.requireTenantId()`, never from the request body.
- `update(UUID id, UpdateItemCategoryRequest)` — never changes `parentId` or `level`.
- `move(UUID id, MoveItemCategoryRequest)` — reject when the new parent is the node itself; reject
  when the new parent is a descendant of the node (walk down from the node collecting descendant ids
  and check membership — this is the cycle guard); reject when the new parent is at level 3; reject
  when moving would push the node's own deepest descendant past level 3 (compute the subtree depth
  and check `newParentLevel + subtreeDepth <= 3`). On success, update `level` on the node AND on
  every descendant in the same transaction so the trigger's parent-level invariant continues to hold.
- `archive(UUID id)` — count non-archived children and count ingredients whose `categoryId` is this
  node; if either is greater than zero throw `CategoryInUseException(id, ingredientCount, childCount)`;
  otherwise set `archivedAt = Instant.now()` and save. Never call a repository delete.
- `restore(UUID id)` — clears `archivedAt`, refusing when the parent is itself archived.

`CategoryInUseException` carries the category id, the ingredient count and the child count. Add a
handler in `InventoryExceptionHandler` mapping it to HTTP 409 with error code `CATEGORY_IN_USE` and a
message matching the UI contract's copy, interpolating the live ingredient count so the dialog can
render the server message directly.
  </action>

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08.2-061 | Elevation of Privilege | Category write endpoints | high | mitigate | `authz.authorizeManage` is the first statement of every mutating handler; `CategoryAccessControlIT` proves a view-only principal is refused. |
| T-08.2-062 | Tampering | Client-supplied `tenantId` on create | high | mitigate | `tenantId` is absent from every request record and always set from `tenantContext.requireTenantId()`; `CategoryAccessControlIT` asserts an injected body field is ignored. |
| T-08.2-063 | Tampering | Cross-tenant `parentId` grafting a node onto another tenant's tree | high | mitigate | The parent is loaded via `findByTenantIdAndId`, and the V5 trigger independently rejects a cross-tenant parent — defence in depth. |
| T-08.2-064 | Denial of Service | Cyclic re-parent causing an unbounded tree walk | medium | mitigate | `move` collects the node's descendant id set first and refuses membership of the new parent; the DB CHECK on `level` bounds depth regardless. |
| T-08.2-065 | Repudiation | Silent loss of GL history by deleting an in-use category | high | mitigate | No delete path exists; archive is refused while any ingredient or non-archived child references the node (D-04). |
</threat_model>
