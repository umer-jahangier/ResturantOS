<objective>
Build the entire inventory frontend data layer for this phase in one place: Zod schemas mirroring the
new backend DTOs, adapter types, repository methods, TanStack hooks on the central key registry, and
MSW handlers realigned with the backend's coverage definition.

Purpose: the four-layer pattern (schema → adapter → repository → hook) is ESLint-enforced and every
screen plan in wave 5 consumes it. Building it once here is what lets six screen plans run in
parallel afterwards without fighting over the same four files. It also closes the mock/backend
coverage-definition mismatch that CONTEXT.md identifies as the reason no test caught the origin bug.
Output: five files, covering categories, ingredients, stock levels, cost preview and three-state
coverage.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API response → Zod parse | Untrusted server payload validated at the schema boundary before reaching any component. |
| Component → repository → gateway | ESLint forbids components importing `lib/api-client` directly, so every request goes through a validated repository method. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08.2-121 | Tampering | Unvalidated API payloads reaching components | medium | mitigate | Every repository method parses the response with its Zod schema and every write parses the input before the call; the four-layer boundary is ESLint-enforced. |
| T-08.2-122 | Information Disclosure | Branch data leaking across a branch switch via stale cache | high | mitigate | All inventory keys are branch-scoped through `queryKeys.inventory.*`, so a branch switch invalidates cleanly the way finance and pos already do. |
| T-08.2-123 | Repudiation | Mock and backend disagreeing so a defect ships untested | high | mitigate | `computeCoverage` in MSW is rewritten to the backend's effective-from definition and a future-dated fixture is added; a grep gate fails the build if the boolean definition returns. |
| T-08.2-124 | Denial of Service | A cost-preview request per keystroke | medium | mitigate | The preview input is debounced at 400ms before it reaches the query key, and `enabled` is false with no lines. |
</threat_model>
