<objective>
Put Java behind the vendor catalog: `VendorItem` (vendor SKU, pack size, purchase UOM, MOQ, lead
time) linked one-to-one to an ingredient, append-only effective-dated `VendorItemPrice`, and
filter-only `VendorCategory` tags.

Purpose: PUR-07 and ROADMAP Success Criterion 5. `vendor_catalogues` has been in the schema since
purchasing V1 with zero Java behind it; this is the entity that lets a PO line be priced, three-way
matched, and compared across vendors.
Output: three entities, three repositories, DTOs, two services, one controller, and two integration
tests ŌĆö one of which exists purely to prove prices are never overwritten.
</objective>

`VendorItemPriceService` (`@Service`, class-level `@Transactional(readOnly = true)`):
- `recordNewPrice(UUID vendorItemId, RecordVendorItemPriceRequest request)` ŌĆö `@Transactional`.
  Resolve `tenantId`; load the vendor item under that tenant (404 otherwise); resolve
  `effectiveFrom` to the request value or `Instant.now()`; load the currently-open price row for the
  same `(vendorItemId, branchId)` scope and set its `effectiveTo` to the new `effectiveFrom`; then
  INSERT a brand-new `VendorItemPrice`. The only field ever written on the prior row is
  `effectiveTo`. Reject an `effectiveFrom` that is at or before the currently-open row's own
  `effectiveFrom` with a 4xx-mapped domain exception, so history cannot be interleaved backwards.
- `listPrices(UUID vendorItemId)` ŌĆö full history, newest first.
- `currentPrice(UUID vendorItemId, Instant at)` ŌĆö the row whose window covers `at`.
- `priceChanges(UUID vendorId, Instant since)` ŌĆö for every non-archived catalog row of the vendor,
  walk its price rows in chronological order and emit a `VendorItemPriceChangeDto` per transition,
  with `deltaPct` computed as BigDecimal at scale 2 HALF_UP and null when there is no previous price.
  Load all prices for the vendor's items in ONE batched query.
