<objective>
Stand up the `user-service` module (servlet stack, port 8082) as the Tenant-Admin surface for branches and per-branch role assignment, and add the auth-service `/internal/auth/**` endpoints that make auth the system of record for `user_branch_roles` and permission computation. user-service owns the tenant-scoped `branches` table (RLS), exposes `/api/v1/branches/**` CRUD + `/api/v1/users/**` role assignment, exposes the internal `POST /internal/users/branches` the provisioning saga calls (FD-1 step 4) + branch-detail reads, and DELEGATES every `user_branch_roles` write to auth-service (never touches that table itself).

Purpose: Delivers SC5 (Tenant Admin CRUDs branches + assigns roles per branch; internal endpoints return branch details + computed permissions for JWT issuance). Covers USER-01 (branch CRUD, tenant-scoped via RLS), USER-02 (user profile/role management per branch), USER-03 (internal branch + permission endpoints feeding JWT issuance). Establishes the `createBranch` endpoint + the auth-service `/internal/auth/**` internal-call security gate that 03-02 (platform-admin provisioning) depends on.
Output: the `user-service` Maven module (+ parent POM line), `branches` Liquibase migration with RLS, Branch entity/repo/service/controllers, an `AuthInternalClient` Feign delegating role writes + permission reads, NEW auth-service `AuthInternalController` (branch-roles + permissions) gated by `InternalServiceFilter`, a one-line gateway route extension to cover `/api/v1/branches/**`, and a Testcontainers IT suite proving RLS isolation + delegation.

OWNERSHIP DECISION (locked ÔÇö option (a) from planning_context): auth-service stays the single system of record for `users` + `user_branch_roles` + permission computation. user-service owns ONLY `branches` and orchestrates the Tenant-Admin surface, delegating role/permission writes+reads to auth-service `/internal/auth/**`. user-service has NO `user_branch_roles` table and NO users table.

BUDGET NOTE: 4 tasks (~19 files) ÔÇö heavier than the 3-task norm because it also adds auth-service internal endpoints + their security gate (a hard prerequisite for 03-02). Task 1 = scaffold + branches migration/RLS + entity/repo. Task 2 = auth-service internal endpoints + InternalServiceFilter gate + auth IT. Task 3 = user-service services/controllers + Feign delegation + internal endpoints + gateway route. Task 4 = user-service IT suite. Natural split points are between every task; do NOT split mid-task. If context pressure hits ~50% after Task 3, stop and let Task 4 run in a fresh window.
</objective>
