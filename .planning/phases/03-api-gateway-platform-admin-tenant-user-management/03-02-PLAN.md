<objective>
Stand up the `platform-admin-service` module (servlet stack, internal port 8096) as the SuperAdmin control plane: provision tenants (FD-1) via a sequential saga with compensation, list/operate tenant lifecycle, own per-tenant feature flags with immediate Redis invalidation, impersonate tenant users, and record usage telemetry — backed by `platform_db`, which is deliberately NOT tenant-scoped (no RLS). It also adds the auth-service internal endpoints provisioning needs (provision-admin, service-token, impersonate), reusing the `/internal/auth/**` gate that 03-03 established.

Purpose: Delivers SC3 (provision < 60s with tier features + admin + HQ branch + TENANT_PROVISIONED; list/paginate; suspend/reactivate/cancel; feature flag update with immediate cache invalidation; impersonate stamped impersonated_by 30-min logged; telemetry), SC4 (platform_db has no tenant_id RLS, only this service connects), and SC6 (tier-independent per-tenant module enable/disable persisted on tenant_features, invalidated immediately, audited, enforced identically at the gateway + @RequiresFeature aspect; six primary modules + KDS default ON). Covers PLATFORM-01..07 and PLATFORM-10.
Output: the `platform-admin-service` Maven module (+ parent POM line), platform_db Liquibase (5 NON-RLS tables + seed platform_users), entities that do NOT extend TenantAuditableEntity, the provisioning saga + lifecycle + feature-flag + impersonation + usage services, public `/api/v1/platform/**` + internal `/internal/platform/**` controllers, three internal Feign clients, the NEW auth-service provisioning/impersonation internal endpoints + a `signImpersonationToken` method, and a Testcontainers IT suite (with WireMock for auth/user/finance).

PORT RESOLUTION (locked — Doc 5 + SaaS spec port table): the gateway owns public 8080; platform-admin-service is an INTERNAL service and takes its own distinct dev port 8096 (Doc 5 platform `8080`/`8096(dev)` column) → `SERVER_PORT: ${SERVER_PORT:8096}`. Doc 4 §4.2's `http://platform-admin-service:8080` is the in-container hostname:port; callers reach it via Eureka `lb://platform-admin-service`, not a hardcoded port, so there is NO collision. Do NOT bind platform-admin to 8080.

Task 1 = scaffold + platform_db NON-RLS migrations + entities/repos/seed + tier defaults. Task 2 = auth-service provision-admin/service-token/impersonate + signImpersonationToken + Feign clients. If context hits ~50% after Task 3, stop and run Task 4 fresh.
