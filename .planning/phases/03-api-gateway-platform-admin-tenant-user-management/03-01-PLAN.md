<objective>
Stand up the `gateway` module (Spring Cloud Gateway 5.x WebFlux, public port 8080) as the platform edge: validate access JWTs and resolve `tenant_id`, inject `X-Tenant-Id`/`X-User-Id` downstream, enforce per-IP rate limits, per-tenant feature flags + tenant status + NLQ quota, and per-upstream circuit breakers — sitting behind an Nginx TLS terminator.

Purpose: Delivers SC1 (invalid JWT → 401, public paths pass, X-Tenant-Id propagated) and SC2 (routing + circuit breakers + rate limits + FEATURE_DISABLED + QUOTA_EXCEEDED + Nginx TLS). Covers GW-01 through GW-06. Establishes the edge contract the rest of the platform routes through.
Output: the `gateway` Maven module (+ parent POM module line), reactive `JwtGlobalFilter` + `FeatureFlagGlobalFilter`, route/rate-limit/circuit-breaker YAML under the NEW `spring.cloud.gateway.server.webflux.*` prefix, a fallback controller, a reactive WebClient to platform-admin for cache-miss feature lookups, an Nginx config, and a WebTestClient/Testcontainers IT suite.

Task 2 = the two reactive GlobalFilters + tenant resolution + fallback + platform-admin WebClient. Task 3 = IT suite.     `RateLimitConfig`: `@Bean KeyResolver ipKeyResolver()` reading `X-Forwarded-For` first token, falling back to `getRemoteAddress()` (RESEARCH Code Examples). This bean name MUST match the `#{@ipKeyResolver}` SpEL in application.yml.

    `GatewaySecurityConfig`: a Spring Security 7 reactive `SecurityWebFilterChain` (`ServerHttpSecurity` lambda DSL — NOT `WebSecurityConfigurerAdapter`, removed in Security 7, RESEARCH State of the Art). Disable CSRF, permitAll for `/actuator/health/**`, `/actuator/prometheus`, `/fallback/**`, and the public auth/jwks paths; everything else `permitAll` at the Security layer too (authentication is enforced by `JwtGlobalFilter`, not Spring Security — the gateway is a router, the filter owns auth). Produce a `JwksKeyProvider` bean configured to fetch auth-service JWKS at `${restaurantos.jwks.uri:http://auth-service:8081/.well-known/jwks.json}` (reuse shared-lib's HTTP/TTL/kid-rotation handling; do NOT re-implement key caching — RESEARCH Don't Hand-Roll).

    `JwtGlobalFilter implements GlobalFilter, Ordered` at `Ordered.HIGHEST_PRECEDENCE + 10` (RESEARCH Pattern 1 — a reactive GlobalFilter; do NOT import shared-lib's servlet `JwtAuthenticationFilter`, Pitfall 3). Logic:
      1. `isPublicPath(path)` → pass through: `/api/v1/auth/login`, `/api/v1/auth/refresh`, `/api/v1/auth/reset-password/**`, `/.well-known/**`, `/actuator/health/**`, `/actuator/prometheus`, `/fallback/**`.
      2. Read `Authorization: Bearer`. Missing/malformed → `writeError(exchange, 401, "UNAUTHENTICATED")` (JSON ApiError body via the response buffer; do NOT call upstream).
      3. Verify RS256 signature + expiry against `JwksKeyProvider` (nimbus `JWSVerifier` keyed by the `kid` header) and parse into `JwtClaims` (shared-lib record — sub, tenant_id, branch_id, roles[], permissions[], attributes{}, impersonated_by?). Any failure → 401 UNAUTHENTICATED.
      4. Resolve tenant via `TenantResolutionSupport` (below). Mutate the request adding `X-Tenant-Id`, `X-User-Id`, and (if present) `X-Impersonated-By` headers, then `chain.filter` with the mutated exchange. (Pitfall 7 — propagate `impersonated_by` so downstream audit can log it.)

    `TenantResolutionSupport`: resolve `tenant_id` PRIMARY from the JWT `tenant_id` claim. SECONDARY (custom-domain/subdomain tenants): if the claim is absent, extract slug from the `Host` header, `Redis GET tenant:slug:{slug}`, and on miss fall back to `PlatformAdminClient` (document that full custom-domain provisioning is a later milestone — RESEARCH Open Q2; keep this path minimal but present). Throw → 401 if unresolvable on a protected path.

    `FeatureFlagGlobalFilter implements GlobalFilter, Ordered` at `Ordered.HIGHEST_PRECEDENCE + 20` (runs AFTER JwtGlobalFilter). Logic (RESEARCH Pattern 3 + FD-15):
      1. Skip for public paths and for routes with no feature mapping.
      2. Reactive `Redis GET tenant:status:{tenantId}`; on miss call `PlatformAdminClient.getStatus(tenantId)` and cache with 5-min TTL (RESEARCH Open Q4). If status != ACTIVE → 403 `TENANT_SUSPENDED` (JSON ApiError).
      3. Map the request path → required `FEATURE_*` via `RouteFeatureMap` (e.g. `/api/v1/hr/**` → `FEATURE_HR`, `/api/v1/crm/**` → `FEATURE_CRM`, `/api/v1/nlq/**` → `FEATURE_NLQ`). For mapped routes: `Redis GET tenant_features:{tenantId}:{code}`, miss → `PlatformAdminClient.getFeatures(tenantId)` cached 5-min. Disabled → 403 `FEATURE_DISABLED` with header `X-Upgrade-CTA-URL: https://app.restaurantos.io/billing?feature={code}` (RESEARCH Code Examples / FD-15).
      4. For quota-bearing routes (NLQ): `Redis GET nlq_quota:{tenantId}:monthly_count` vs limit; over → 403 `QUOTA_EXCEEDED`. (Counter increments are owned by the NLQ service later; gateway only reads/enforces here — document the seam.)

    `RouteFeatureMap`: a small `@Component` mapping path prefixes → feature codes (Appendix C codes). Keep it data-driven (a `Map<String,String>` or ordered list of prefix→code).

    `PlatformAdminClient`: a reactive `WebClient` bean targeting `lb://platform-admin-service` (or `${platform-admin.uri}`) calling `GET /internal/platform/tenants/{id}/status` and `GET /internal/platform/tenants/{id}/features` (Doc 4 §4.2). These are the cache-miss fallbacks. NOTE: platform-admin-service is built in 03-02; until then these calls fail closed — wrap in `onErrorResume` that, on connection failure, treats the tenant as ACTIVE with no feature restrictions ONLY in a `gateway.fail-open-on-platform-down` dev profile, and fail-closed (403) in prod. Document this seam clearly in the SUMMARY.

    `FallbackController` (`@RestController`): `@RequestMapping("/fallback/service-unavailable")` returning 503 with JSON `ApiError{code:"SERVICE_UNAVAILABLE"}` (target of the CircuitBreaker `fallbackUri`).
