<objective>
Add the self-service provision endpoint `POST /api/v1/finance/periods/provision` (FIN-08) so an OWNER/TENANT_ADMIN/ACCOUNTANT can (re-)provision their own tenant's CoA + accounting periods — the self-heal path for a tenant whose onboarding finance step failed (plan 03) or whose fiscal year has rolled over. Per RESEARCH.md this is a thin controller wrapper around the already-idempotent `ProvisioningService.provision(tenantId, fiscalYear)` — no new business logic, no new service method, no new DTO. Security-critical detail: the endpoint MUST resolve tenantId from the JWT-derived `TenantContext`, never from a path variable or request body (that `@PathVariable tenantId` shape is reserved for `/internal/**` endpoints).

Purpose: Give tenant admins an idempotent, permissioned way to guarantee an open period without platform-ops touching internal APIs.
Output: A new `@PreAuthorize`-gated controller method + integration tests for happy-path and idempotent repeat.
</objective>
