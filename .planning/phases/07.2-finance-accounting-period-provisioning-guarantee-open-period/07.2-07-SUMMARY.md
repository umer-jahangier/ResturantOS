## Decisions Made
- `ProvisionPeriodDialog.tsx` uses a local `getProvisionErrorMessage()` (mirrors `components/pos/payment-panel.tsx`'s `getChargeErrorMessage` convention) instead of `formatUserFacingError` from `@/lib/api-client/errors` — the latter is a documented, pre-existing ESLint layer-boundary violation in sibling components (`docs/finance-eslint-backlog.md` Issue 1); since this plan's own acceptance criteria require the targeted `eslint` run to pass clean, the new file avoids reintroducing that violation rather than following `PeriodCloseModal.tsx` byte-for-byte on this one point.
- `ProvisionPeriodDialog`'s internal `fiscalYear` state resets via a parent-side `key={fiscalYear}` remount in `periods/page.tsx`, not a `useEffect`+`setState` sync — required by this repo's `react-hooks/set-state-in-effect` ESLint rule (same constraint documented in prior phases' STATE.md decisions, e.g. 04-04-A, 07-03-C).
- The E2E spec's `login()` helper explicitly classifies a "Sign-in failed / service temporarily unavailable" banner as `Blocked` (environment/backend-down precondition), matching `pos-settlement.spec.ts`'s existing convention for its own 503/FallbackController cases — discovered necessary during this plan's own live-run attempt (see Deviations).

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Avoided a layer-boundary ESLint violation in the new dialog component**
- **Found during:** Task 2 (`pnpm eslint` verification step)
- **Issue:** The plan's `<action>` directs mirroring `PeriodCloseModal.tsx`'s exact error-display pattern, which imports `formatUserFacingError` from `@/lib/api-client/errors` — a `components/**` file importing `lib/api-client/**` directly. This IS the project's enforced `no-restricted-imports` boundary (FE-08) and is a pre-existing, documented violation in `PeriodCloseModal.tsx`/`JournalEntryForm.tsx` (`docs/finance-eslint-backlog.md`). Copying it into the NEW `ProvisionPeriodDialog.tsx` would fail this plan's own acceptance criteria ("targeted eslint run passes").
- **Fix:** Added a small local `getProvisionErrorMessage(error: unknown): string` in `ProvisionPeriodDialog.tsx`, following the same in-repo convention already used by `components/pos/payment-panel.tsx`'s `getChargeErrorMessage` (a component-local formatter, no `api-client` import).
- **Files modified:** `frontend/components/finance/ProvisionPeriodDialog.tsx`
- **Verification:** `pnpm eslint components/finance/ProvisionPeriodDialog.tsx` — 0 errors (previously 1 `no-restricted-imports` error with the plan's literal instruction)
- **Committed in:** `c2f89cd` (Task 2 commit)

**2. [Rule 3 - Blocking] E2E login helper needed to classify a backend-down banner as Blocked, not FAIL**
- **Found during:** Task 3's live-run verification step (`npx playwright test ...`)
- **Issue:** The dev stack's `finance-service` process was not running this session (confirmed: `services/finance-service` port 8086 unreachable; gateway `/actuator/health` on 8080 returned 503). Login for both `accountant@demo.local` and `cashier@demo.local` surfaced a "Sign-in failed / The service is temporarily unavailable" banner.
