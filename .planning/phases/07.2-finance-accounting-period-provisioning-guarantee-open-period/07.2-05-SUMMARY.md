# Phase 07.2 Plan 05: Self-service accounting-period provision endpoint Summary

**Added `POST /api/v1/finance/periods/provision` — a `@PreAuthorize`-gated, JWT-tenant-scoped, idempotent self-service endpoint that lets OWNER/TENANT_ADMIN/ACCOUNTANT (re-)provision their own tenant's CoA + accounting periods, delegating with zero new business logic to the existing `ProvisioningService.provision(tenantId, fiscalYear)` (FIN-08).**

## Performance

- **Duration:** 20 min
- **Tasks:** 2
- **Files modified:** 3 modified, 1 created

## Accomplishments

- `PeriodController` gained a new `POST /api/v1/finance/periods/provision` method, gated `@PreAuthorize("hasAuthority('finance.period.open')")`, injected with `ProvisioningService` + `io.restaurantos.shared.tenant.TenantContext` via the extended constructor.
- tenantId is resolved exclusively from `tenantContext.requireTenantId()` — the method signature has no `@PathVariable`/body tenantId parameter, so cross-tenant provisioning is structurally impossible (threat T-07.2-05-01 mitigated).
- `fiscalYear` defaults to `PakistanFiscalYear.current()` when the optional `ProvisionRequest` body is omitted, and is validated to a plausible range (`2000` to `current()+5`); out-of-range values throw a new `InvalidFiscalYearException`, mapped by a new `FinanceGlobalExceptionHandler.handleInvalidFiscalYear` to `400 INVALID_FISCAL_YEAR` (threat T-07.2-05-03 mitigated; no ad-hoc try/catch/Map in the controller).
- Endpoint delegates directly to `provisioningService.provision(tenantContext.requireTenantId(), fiscalYear)` and returns `ResponseEntity.ok(ApiResponse.ok(...))` — no new service method, no new DTO (reuses existing `ProvisionRequest`/`ProvisioningResult`).
- `AccountingPeriodIT` extended with two new tests: `provisionEndpoint_seedsPeriodsForCurrentTenant()` (asserts `periodsSeeded() == 12` and 12 rows for the current fiscal year) and `provisionEndpoint_idempotentOnRepeat()` (asserts the second identical call seeds `0` with the row count staying at 12). Full suite: 8/8 green (6 pre-existing + 2 new).
- Blast-radius check (grep, GitNexus MCP unavailable in this session): confirmed no pre-existing route collision on `/api/v1/finance/periods/provision` and no other `PeriodController` definition before adding the method.

## Decisions Made

- Created `InvalidFiscalYearException` (new file) rather than reusing `IllegalStateException`'s generic 400 handler, so the error carries a distinct `INVALID_FISCAL_YEAR` code and mirrors the existing `InvalidAccountCodeException` convention exactly.
- Plausibility range chosen as `[2000, PakistanFiscalYear.current()+5]` — generous enough to never block legitimate onboarding/rollover use, narrow enough to reject obviously garbage values (negative, zero, far-future).
- Integration tests exercise `provisioningService.provision(...)` directly (the endpoint's documented exact delegate) instead of the `PeriodController` bean, after confirming a direct bean call fails under Spring's method-security AOP with no populated `SecurityContext` — this was anticipated by the plan's own context note and required no plan deviation, just choosing the delegate-call option the plan explicitly offered.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Corrected test invocation layer to avoid @PreAuthorize AuthenticationCredentialsNotFoundException**
- **Found during:** Task 2
- **Issue:** Initial test draft autowired `PeriodController` and called `provisionPeriods(null)` directly. Spring's method-security AOP interceptor enforces `@PreAuthorize` on every bean invocation (not only HTTP-routed calls), so this threw `AuthenticationCredentialsNotFoundException: An Authentication object was not found in the SecurityContext` — confirmed by running the suite.
- **Fix:** Switched both new tests to call `provisioningService.provision(tenantId, fiscalYear)` directly — the endpoint's exact delegate, as explicitly permitted by the plan's task action ("invoke the new controller method ... or its exact delegate").
- **Files modified:** `services/finance-service/src/test/java/io/restaurantos/finance/AccountingPeriodIT.java`
- **Commit:** `b5f9021`
