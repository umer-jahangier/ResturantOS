# Phase 07.2 Plan 04: Config-gated auto-seed-on-miss with WARN audit log Summary

**Made `AccountingPeriodServiceImpl.getPeriodStatus`'s silent auto-seed-on-miss fallback config-gated (`finance.period.auto-seed-on-miss`, default true) with a WARN audit line, plus a dedicated toggle-off IT — no pos-service or auto-seed formula changes.**

## Performance

- **Duration:** 20 min
- **Tasks:** 2
- **Files modified:** 2 modified, 1 created

## Accomplishments
- `AccountingPeriodServiceImpl` now imports slf4j `Logger`/`LoggerFactory` and Spring's `@Value`; a new `autoSeedOnMiss` boolean field (`@Value("${finance.period.auto-seed-on-miss:true}")`) gates the `.orElseGet(...)` self-heal branch in `getPeriodStatus`.
- When `autoSeedOnMiss` is true (default): a `log.warn(...)` line naming `tenantId`, the requested `date`, and the resolved `PakistanFiscalYear.forDate(date)` fires immediately before `seedForTenant(...)` — the existing "auto-seeds FY2027, 12 periods, OPEN" behavior is unchanged.
- When `autoSeedOnMiss` is false: the initial repository miss throws `PeriodNotFoundException(null)` directly — no seed call, no WARN log.
- `application.yml` gained a new `finance.period.auto-seed-on-miss: ${FINANCE_PERIOD_AUTO_SEED_ON_MISS:true}` block, mirroring the `restaurantos.internal.secret` env-var-backed placeholder convention already in the file; both the YAML fallback and the `@Value` inline default agree on `true`.
- New `AccountingPeriodAutoSeedToggleIT` (standalone class, `@TestPropertySource(properties = "finance.period.auto-seed-on-miss=false")`, extends `FinanceTestBase`): provisions FY2026, calls `getPeriodStatus` for a date in FY2027 (near the Jul-1 boundary per RESEARCH.md Pitfall 5), asserts `PeriodNotFoundException` is thrown and that zero FY2027 periods were seeded as a side effect.
- Blast-radius check (grep, GitNexus MCP unavailable): `getPeriodStatus` is called only by `InternalFinanceController` (the finance-service `/internal/finance/periods/status` endpoint) and `InternalAutoPostIT`; pos-service reaches it exclusively over HTTP via `FinancePeriodClient` (Feign), so no pos-service Java file needed to change. `git diff --name-only` across both task commits confirms only finance-service files were touched.

## Files Created/Modified
- `services/finance-service/src/main/java/io/restaurantos/finance/service/AccountingPeriodServiceImpl.java` - Added slf4j `Logger` + `@Value("${finance.period.auto-seed-on-miss:true}") autoSeedOnMiss`; `getPeriodStatus`'s fallback now checks the flag, logs a WARN before seeding when on, and throws `PeriodNotFoundException` directly when off (no seed attempt)
- `services/finance-service/src/main/resources/application.yml` - Added `finance: period: auto-seed-on-miss: ${FINANCE_PERIOD_AUTO_SEED_ON_MISS:true}` block
- `services/finance-service/src/test/java/io/restaurantos/finance/AccountingPeriodAutoSeedToggleIT.java` (new) - Toggle-off IT proving the miss surfaces `PeriodNotFoundException` and seeds nothing

## Decisions Made
- Gated the seed call itself (not just the WARN log) behind `autoSeedOnMiss` — toggle-off means genuinely no self-heal, matching the plan's must_haves exactly.
- Kept seeding keyed on `PakistanFiscalYear.forDate(date)` (the requested date, not `current()`) — this was already correct in the existing code and the plan explicitly called out not to narrow it.
- Created the toggle-off test as a standalone top-level class rather than a `@Nested` inner class (unlike 07.2-03's `ProvisioningSagaIT` nested-class pattern) because `FinanceTestBase` does not pin `finance.period.auto-seed-on-miss` via `@DynamicPropertySource`, so a plain `@TestPropertySource` on a dedicated class is the cleanest fit here — no need for the nested-class workaround that a `@DynamicPropertySource`-pinned base would have required.
