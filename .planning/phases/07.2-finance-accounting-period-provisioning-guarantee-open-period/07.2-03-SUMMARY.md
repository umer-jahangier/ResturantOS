# Phase 07.2 Plan 03: Onboarding Saga Finance-Seed Hardening Summary

**Removed the inner try/catch WARN-swallow around `financeClient.seedCoa()` in `ProvisioningService.provision()` Step 5 and flipped `provisioning.seed-coa.enabled`'s YAML default from `false` to `true`, so a finance-seeding failure now aborts onboarding (PROVISIONING_FAILED) instead of producing an ACTIVE tenant with zero accounting periods — the root cause of the 423 PERIOD_LOCKED blocker.**

## Accomplishments

- `ProvisioningService.provision()` Step 5 is now a fail-fast saga step: `financeClient.seedCoa(tenantId)` sits directly inside the `if (seedCoaEnabled)` guard with no inner catch, so its exception propagates to the saga's existing outer `catch (Exception ex)` block (which already sets `TenantStatus.PROVISIONING_FAILED`, runs compensation, and rethrows `ProvisioningException`).
- `provisioning.seed-coa.enabled`'s effective default is now `true` everywhere: the `@Value("${provisioning.seed-coa.enabled:true}")` inline default was already `true`; `application.yml`'s env-var fallback (`${PROVISIONING_SEED_COA_ENABLED:false}` → `${PROVISIONING_SEED_COA_ENABLED:true}`) was the actual live default and is now flipped to agree.
- `BasePlatformIT` gained `stubFinanceSeedCoaFail()` (WireMock 500 on `POST /internal/finance/tenants/{tenantId}/seed-coa`, path-matched since the tenant ID isn't known until the saga runs) and the hardcoded `provisioning.seed-coa.enabled=false` line was removed from its `@DynamicPropertySource` — the `false` default now comes solely from `application-test.yml`, and is per-class overridable.
- `ProvisioningSagaIT` gained a new `@Nested FinanceSeedFailureTests` class (its own `@TestPropertySource(properties = "provisioning.seed-coa.enabled=true")`) with `provisionTenant_financeSeedFails_compensates_tenantProvisioningFailed()`, proving the fix end-to-end: branch-create + admin-provision stubbed to succeed, finance-seed stubbed to 500 → `provisioningService.provision(...)` throws `ProvisioningService.ProvisioningException` and the persisted tenant ends `PROVISIONING_FAILED`.
- TDD gates confirmed: RED (test failed against the unhardened Step 5 — WARN logged, no exception thrown, `AssertionError: Expecting code to raise a throwable`) then GREEN (full `ProvisioningSagaIT` class, 4/4 tests, including the new one).
