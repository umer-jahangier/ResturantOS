## Sampling Rate

- **After every task commit:** Run `mvn -pl <touched-service> test` (backend, unit-level only, fast) or `pnpm vitest run <file>` + `pnpm tsc --noEmit` (frontend)
- **After every plan wave:** Run `mvn -pl services/finance-service,services/platform-admin-service,services/auth-service -am verify` (Wave 1); `pnpm tsc --noEmit && pnpm eslint <touched files>` (Wave 2, frontend); the live checkpoint (Wave 3)
- **Before `/gsd-verify-work`:** Full backend suite must be green, PLUS a live restart-and-manual-verify pass against the dev stack (a code fix in a jar that isn't running is not a fix — see Pitfall 3 in RESEARCH.md) — the frontend needs no restart (Next.js Fast Refresh) but its Playwright spec's live pass/fail is only meaningful AFTER that backend restart
- **Max feedback latency:** ~360 seconds (full IT suite)
