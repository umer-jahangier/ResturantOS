<objective>
Prove the fix is actually live, not just committed. This is the phase gate. RESEARCH.md documented that the finance-service process was running a stale jar during Phase 07.1's E2E — a code fix in a jar that isn't running is not a fix (Pitfall 3), and this exact class of failure already bit the project. So before any live/E2E claim, restart every service this phase touched onto its current jar and confirm `/actuator/health` UP. Then run the full IT suite, then drive the real reported bug (POS order-close 423 on a fresh/period-less tenant) to resolution, and confirm the new permission gate behaves live.

Purpose: End-to-end confirmation that FIN-07 (saga hardening), FIN-08 (self-service endpoint + permission), and FIN-09 (config-gated auto-seed) resolve the 423 PERIOD_LOCKED blocker in the running dev stack.
Output: A verification SUMMARY with restart/health evidence, the full IT-suite result, live E2E evidence (provision endpoint and/or Playwright order-close), and the permission-gate result.
</objective>
