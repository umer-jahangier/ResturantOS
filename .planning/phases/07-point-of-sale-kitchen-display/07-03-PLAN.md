<objective>
Make the POS terminal offline-first (POS-07): a Workbox PWA that precaches the app shell + menu/table reference data, an IndexedDB outbox that captures order create/append while offline using a client-generated `client_order_id`, and a sync engine that replays FIFO on reconnect against the 07-01 idempotent endpoint so replays never duplicate. Money-/authorization-sensitive actions (close, payment, void, refund, till) stay online-only and fail-safe.

Purpose: a dropped connection must never lose an order or take a duplicate; this builds directly on the `client_order_id` server idempotency from 07-01.
Output: PWA + service worker, IndexedDB outbox + sync engine with dedupe reconciliation, online/offline + pending-sync UI, and tests (unit + e2e) proving offline create -> reconnect -> exactly one order.
</objective>

  <files>

  <action>
**sync-engine.ts:**
- `replay()`: drain `outbox` FIFO. For each PENDING op -> markInFlight -> call `pos.repository` (CREATE_ORDER/APPEND_ITEMS) with the op's `clientOrderId` as the idempotency field/header (07-01 server dedupes).
- **Reconciliation (idempotent/resumable):** treat both "201 created" and the server's "already exists for this client_order_id" (200 with existing order, or 409/idempotent-replay response) as SUCCESS -> markSynced + remove. A replay must NEVER create a second order.
- **Backoff retry:** on transient/network error -> markFailed with incremented attempts + exponential backoff; re-attempt on next trigger. On 4xx validation (non-idempotent) -> markFailed terminal + surface (don't infinite-loop).
- **Triggers:** on `online` event, on app focus, and a periodic interval; guard against concurrent replays (single-flight lock).
- Emit progress (pending count, last error) for the UI badge.

**sync-engine.test.ts** (fake-indexeddb + mocked repository):
- offline create -> one outbox op; replay online -> repository called once with clientOrderId -> op SYNCED; outbox empty.
- duplicate replay (server returns existing order for same clientOrderId) -> reconciled as success, NO second create, op removed.
- transient failure -> op stays, attempts incremented, retried next run -> succeeds.
- FIFO across multiple ops preserved.
