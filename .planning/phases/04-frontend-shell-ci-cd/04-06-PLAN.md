  <action>
    **`components/ui/command-palette.tsx`** (`'use client'`):
    - Import `Command` from `cmdk` (the `cmdk` package provides `<Command>`, `<Command.Input>`, `<Command.List>`, `<Command.Group>`, `<Command.Item>`, `<Command.Empty>`, `<Command.Separator>`)
    - Create a `<CommandPalette>` that:
      - Accepts `open: boolean`, `onOpenChange: (open: boolean) => void`, and `children: React.ReactNode` (for command items)
      - Renders a `<Dialog>` (from existing shadcn ui or a simple portal overlay) with the `<Command>` inside
      - Has a search input with placeholder "Type a command or search…"
      - Shows `<Command.Empty>` with "No results found." when filter returns nothing
      - Registers a global keydown listener for ⌘K / Ctrl+K to toggle open (use `useEffect`; cleanup on unmount)
      - Has smooth entry animation via `scaleIn` variant class or CSS
      - Uses `role="dialog"`, `aria-label="Command palette"`
    - Export as named `CommandPalette`
    - Also export `CommandItem`, `CommandGroup`, `CommandSeparator` re-exports from cmdk for consumer convenience

    **`components/ui/animated-number.tsx`** (`'use client'`):
    - Import `CountUp` from `react-countup`
    - Accept props: `value: number`, `prefix?: string` (e.g., "PKR"), `suffix?: string`, `decimals?: number` (default 0), `duration?: number` (default 1.5), `className?: string`
    - Render `<CountUp end={value} prefix={prefix} suffix={suffix} decimals={decimals} duration={duration} useEasing enableScrollSpy scrollSpyOnce />`
    - Wrap in a `<span className={cn("tabular-nums", className)}>` for consistent number alignment
    - Export as named `AnimatedNumber`

    **`components/ui/status-badge.tsx`:**
    - Accept `status: 'active' | 'inactive' | 'pending' | 'error' | 'warning' | 'success'` and optional `label?: string`, `className?: string`
    - Map statuses to Tailwind classes using the semantic tokens from 04-04:
      - `active` / `success` → `bg-success/15 text-success-foreground border-success/30` (or a visible bg that passes contrast)
      - `error` → `bg-destructive/15 text-destructive-foreground border-destructive/30`
      - `warning` → `bg-warning/15 text-warning-foreground border-warning/30`
      - `info` / `pending` → `bg-info/15 text-info-foreground border-info/30`
      - `inactive` → `bg-muted text-muted-foreground border-border`
    - Render as `<span>` with rounded-full, text-xs, px-2.5 py-0.5, font-medium, border
    - If `label` provided, use it as text; otherwise capitalize the status string
    - Export as named `StatusBadge`
    - No `any`.
  </action>

<task type="auto">
  <name>Task 2: MoneyDisplay + DataTable + EmptyState</name>
  <files>frontend/components/ui/money-display.tsx, frontend/components/ui/data-table.tsx, frontend/components/ui/empty-state.tsx</files>
  <action>
    **`components/ui/money-display.tsx`:**
    - A server-compatible component (NO `'use client'` needed — pure display)
    - Accept `paisa: number | bigint` (BIGINT-safe), optional `currency?: string` (default "PKR"), optional `className?: string`
    - Internally convert paisa to display: `const rupees = Number(BigInt(paisa) / 100n)` and `const paisaPart = Number(BigInt(paisa) % 100n)` — integer math only, NO floating-point division
    - Format using `new Intl.NumberFormat('en-PK', { style: 'currency', currency, minimumFractionDigits: 2, maximumFractionDigits: 2 }).format(rupees + paisaPart / 100)` — the final `/100` for display formatting after integer extraction is acceptable since it's display-only on already-extracted integers
    - Render in `<span className={cn("tabular-nums font-medium", className)}>` for consistent table alignment
    - Export as named `MoneyDisplay`
    - DO NOT accept `amount` as a float. The prop is `paisa` (integer). This enforces §18 Rule 2 ("Paisa stays server-side" — the component receives already-server-validated BIGINT).

    **`components/ui/data-table.tsx`** (`'use client'`):
    - Generic: `DataTable<TData>` where `TData` is the row type
    - Accept props:
      ```ts
      interface DataTableProps<TData> {
        columns: ColumnDef<TData, unknown>[];
        data: TData[];
        isLoading?: boolean;
        emptyMessage?: string;
        pageSize?: number;
      }
      ```
    - Use `@tanstack/react-table` `useReactTable` with `getCoreRowModel`, `getSortedRowModel`, `getPaginationRowModel`
    - If `isLoading`, render `<DataTableSkeleton columns={columns.length} />` from 04-05
    - If `data.length === 0` and not loading, render `<EmptyState title={emptyMessage ?? "No data"} />` (from this same plan)
    - Render a `<table>` with proper `<thead>` / `<tbody>` semantics, Tailwind styling matching the existing shadcn `Table` component style if available
    - Sort indicators: up/down chevron icons from lucide-react on sortable column headers
    - Pagination: prev/next buttons + page info at bottom
    - Re-export `ColumnDef` from `@tanstack/react-table` for consumer convenience
    - No `any` — the generic `TData` flows through

    **`components/ui/empty-state.tsx`:**
    - Accept `icon?: LucideIcon` (from lucide-react), `title: string`, `description?: string`, `action?: { label: string; onClick: () => void }`, `className?: string`
    - Render centered vertically: icon (if provided, 48px, muted-foreground), title (text-lg font-semibold), description (text-sm text-muted-foreground), action button (if provided — uses the existing Button component from shadcn)
    - Default icon: `Inbox` from lucide-react if none passed
    - Export as named `EmptyState`
    - Server-compatible (no 'use client') unless the action button requires it — if so, add 'use client' and document why
  </action>
