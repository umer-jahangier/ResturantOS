<task type="auto">
  <name>Task 2: PageTransition + shared motion variants</name>
  <files>frontend/components/shared/page-transition.tsx, frontend/lib/motion/variants.ts</files>
  <action>
    **`lib/motion/variants.ts`:**
    Export typed framer-motion `Variants` objects. No `any`. All objects are `const`:
    ```ts
    import type { Variants } from "framer-motion";

    export const fadeSlideUp: Variants = {
      initial: { opacity: 0, y: 8 },
      animate: { opacity: 1, y: 0, transition: { duration: 0.35, ease: [0.25, 0.1, 0.25, 1] } },
      exit: { opacity: 0, y: -4, transition: { duration: 0.2 } },
    };

    export const slideInRight: Variants = {
      initial: { opacity: 0, x: 16 },
      animate: { opacity: 1, x: 0, transition: { duration: 0.3, ease: "easeOut" } },
      exit: { opacity: 0, x: -8, transition: { duration: 0.15 } },
    };

    export const scaleIn: Variants = {
      initial: { opacity: 0, scale: 0.95 },
      animate: { opacity: 1, scale: 1, transition: { duration: 0.15, ease: "easeOut" } },
      exit: { opacity: 0, scale: 0.95, transition: { duration: 0.1 } },
    };

    export const staggerContainer: Variants = {
      animate: { transition: { staggerChildren: 0.05 } },
    };
    ```
    Durations match §9 micro-interactions catalogue intent. Ease curve is Material-style.

    **`components/shared/page-transition.tsx`** (`'use client'`):
    - Import `AnimatePresence`, `motion` from `framer-motion` and `fadeSlideUp` from `lib/motion/variants`
    - Accept `children: React.ReactNode` and optional `className?: string`
    - Check `prefers-reduced-motion` via `useReducedMotion()` from framer-motion. If reduced, render children directly without animation.
    - Otherwise wrap children in `<AnimatePresence mode="wait"><motion.div variants={fadeSlideUp} initial="initial" animate="animate" exit="exit" className={className}>{children}</motion.div></AnimatePresence>`
    - Export as `PageTransition`
    - Do NOT place `<AnimatePresence>` around the layout's entire children (that would break Next.js streaming). This is for wrapping a specific page-content section INSIDE a client boundary — the consumer picks where to put it.

    Write Vitest tests:
    - `lib/motion/__tests__/variants.test.ts`: Verify each exported variant has `initial`, `animate`, `exit` keys; verify fadeSlideUp.animate.opacity is 1. Pure unit test, no DOM needed.
    - Total coverage for the variants file should be ≥60%.
  </action>
  <verify>`pnpm --dir frontend exec tsc --noEmit` passes; `pnpm --dir frontend test -- --reporter=verbose variants` passes; `grep -q "useReducedMotion" frontend/components/shared/page-transition.tsx`; `grep -q "AnimatePresence" frontend/components/shared/page-transition.tsx`; `pnpm --dir frontend run lint` passes.</verify>
