  <action>
    **`lib/theme/palette-generator.ts`:**
    - Import `Color` from `colorjs.io` and `wcagContrastCheck` from `./wcag-validator`
    - Export interface `PaletteScale`: `Record<50 | 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900 | 950, string>` (OKLCH string values)
    - Export interface `ThemePalette`: `{ primary: PaletteScale; foreground: string; background: string; contrastValid: boolean }`
    - Export `generatePalette(primaryHex: string): ThemePalette`:
      1. Parse `primaryHex` into `colorjs.io` Color: `const base = new Color(primaryHex).to('oklch')`
      2. Generate 11-stop scale by varying lightness (L channel in OKLCH):
         - 50: L = 0.97, C = base.c * 0.1
         - 100: L = 0.93, C = base.c * 0.2
         - 200: L = 0.87, C = base.c * 0.4
         - 300: L = 0.78, C = base.c * 0.6
         - 400: L = 0.68, C = base.c * 0.8
         - 500: L = base.l, C = base.c (the input colour)
         - 600: L = base.l * 0.85, C = base.c
         - 700: L = base.l * 0.7, C = base.c * 0.9
         - 800: L = base.l * 0.55, C = base.c * 0.8
         - 900: L = base.l * 0.4, C = base.c * 0.7
         - 950: L = base.l * 0.25, C = base.c * 0.6
         All keep hue (H) from base. Clamp L to [0, 1] and C to [0, 0.4].
      3. Serialize each stop as `oklch(L C H)` string
      4. Derive foreground: white for dark backgrounds (500+), black for light backgrounds (50-400). Use `wcagContrastCheck` to validate the 500 (primary) against white and black, pick whichever passes AA.
      5. Return `{ primary: scale, foreground, background: scale[50], contrastValid: wcagContrastCheck(fg, scale[500]).passAA }`
    - No `any`. Pure function, no side effects.

    **`app/api/theme/route.ts`:**
    - Export `GET(request: NextRequest)`:
      1. Extract `brandColor` from `request.nextUrl.searchParams.get('brandColor')`
      2. If missing, return `NextResponse.json({ error: 'brandColor required' }, { status: 400 })`
      3. Validate it looks like a hex colour (`/^#[0-9a-fA-F]{6}$/`)
      4. Call `generatePalette(brandColor)` — import from `lib/theme/palette-generator`
      5. Build CSS string:
         ```
         :root {
           --primary: ${palette.primary[500]};
           --primary-foreground: ${palette.foreground};
           /* ...all 11 scale stops as --primary-50 .. --primary-950... */
         }
         .dark {
           --primary: ${palette.primary[400]};
           --primary-foreground: ${palette.foreground};
         }
         ```
      6. Return `new NextResponse(css, { headers: { 'Content-Type': 'text/css', 'Cache-Control': 'public, max-age=3600' } })`
    - Document in a JSDoc comment the contract: **Backend API contract (Phase 7):** `GET /api/v1/tenants/:id/theme` → `{ brandColor: string, logoUrl: string | null }`. This route will be replaced by a proxied call once the backend delivers tenant settings. For now, the color comes from the query param, and persistence is client-side (localStorage).

    - Test that `contrastValid` is true for standard bright colours
    - Test edge case: very dark colour (e.g., '#1a1a1a') — lightness derivation doesn't produce negative values
    - Test edge case: very light colour (e.g., '#f8f8f8') — scale still spans a usable range
    - Coverage target ≥60% on palette-generator.ts
