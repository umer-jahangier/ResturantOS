<objective>
Add the three list endpoints the UI needs and that do not exist: PO list, vendor-invoice list, expense list.

Purpose: the UAT code audit found 13 of 14 purchasing user journeys unreachable. Two of the biggest blockers are structural, not cosmetic — `purchase-orders/[id]` and `invoices/[id]` have ZERO inbound links because there is nowhere to link FROM: the backend only exposes `GET /{id}`. You cannot build a PO list page against an API that has no list. Same for invoices and (for FIN-05's expense journeys) expenses. This plan is the backend prerequisite for 10-12, 10-13 and 10-14.

Output: three tenant/branch-scoped, permission-gated list endpoints with ITs.
</objective>
