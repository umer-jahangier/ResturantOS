<objective>
Fix Bug 3 — the frontend does not reflect the RBAC the backend enforces.

`frontend/app/(tenant)/app/purchasing/layout.tsx` wraps the module in `<FeatureGuard feature="FEATURE_VENDOR">`
and NOTHING else. `frontend/components/shared/sidebar-nav-items.ts` (L78-84 and L199-206) carries the comment
`// RBAC: vendor.view (see 10-09)` but never sets a `permission:` field — every other nav entry does.

Result: a cashier sees the Purchasing sidebar link, the landing page, all 5 tabs, and on Vendors an empty
state with a live, clickable "Add vendor" button. The backend correctly 403s every call, so nothing leaks —
but the UI advertises a module the user cannot use. Plan 10-09 gated the backend and never brought the
frontend along. This is precisely the class of defect the phase was reopened to eliminate.

Finance already does it correctly: `PermissionGuard require="finance.journal.view"` wrapping `FeatureGuard`.
Copy that shape exactly.

Purpose: make the UI tell the truth about what the user can do.
Output: guarded purchasing layout, permission-carrying nav entries, and a guard so the next module cannot
ship feature-gated-only.
</objective>
