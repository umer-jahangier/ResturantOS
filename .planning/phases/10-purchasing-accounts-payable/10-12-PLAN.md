<objective>
Build the PO user journeys — list, create, submit, withdraw, approve, reject, send — and make partial receipt actually expressible.

Purpose: UAT tests 2, 3, 12 and 13 are all blocked on this. `purchase-orders/[id]` exists but has ZERO inbound links (reachable only by pasting a UUID). There is no PO list page, no create form, and no frontend consumer for submit/withdraw/approve/reject/send at any layer. And `MockGrnReceivePanel` applies ONE qty input to EVERY line (`po.lines.map(l => ({poLineId: l.id, receivedQty: qty}))`), so a genuine partial receipt cannot be expressed — which makes PARTIALLY_RECEIVED, and therefore the entire short-close path 10-04 built, unreachable from the UI.

Output: a complete PO journey through the strict four-layer stack, with per-line receipt inputs.
</objective>

   - `createPurchaseOrderInputSchema` — `{ vendorId: uuid, branchId: uuid, expectedDate: iso date string,
     lines: [{ ingredientId: uuid, description: string, qty: string, unitPricePaisa: string }] }`

   `listPurchaseOrders(branchId, status?)`, `createPurchaseOrder(input)`, `submitPurchaseOrder(id)`,
   `withdrawPurchaseOrder(id)`, `approvePurchaseOrder(id)`, `rejectPurchaseOrder(id, reason)`,
