# Phase 07.3 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.
> Two stacks: backend (pos/kitchen/finance services) and frontend (Next.js).

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework (backend)** | JUnit 5 + Spring Boot Test + Testcontainers (per Phase-7.1 IT pattern) |
| **Framework (frontend)** | Vitest + Testing Library (unit) · Playwright (E2E, `frontend/e2e`) |
| **Config file** | `services/*/pom.xml` · `frontend/vitest.config.ts` · `frontend/playwright.config.ts` |
| **Quick run (backend)** | `mvn -q -pl services/pos-service -am test` (module under change) |
| **Quick run (frontend)** | `pnpm --dir frontend vitest run <changed>` |
| **Full suite command** | backend: `mvn -q test` · frontend: `pnpm --dir frontend test && pnpm --dir frontend exec playwright test` |
| **Estimated runtime** | quick ~30–90 s · full backend ~several min · Playwright ~2–5 min |

---

## Sampling Rate

- **After every task commit:** Run the module's quick command (backend module test OR changed Vitest specs).
- **After every plan wave:** Run the full suite for the touched stack(s); Playwright for any UI-behavior wave.
- **Before `/gsd-verify-work`:** Full suite green + Playwright flows for POS-16, POS-20, POS-21, POS-22/23, KDS-04 green.
- **Max feedback latency:** ~90 s (quick), full suite reserved for wave boundaries.
