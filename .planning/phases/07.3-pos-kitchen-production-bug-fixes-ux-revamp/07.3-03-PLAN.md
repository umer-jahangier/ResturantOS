<objective>
Make the order-taking terminal hold a client-only cart and persist lazily (POS-16/D-01), merge
repeated taps into quantity lines with ± controls (POS-17/D-02), support Dine-in/Takeaway/Pickup
with an optional searchable table selector (POS-18/D-03), and add explicit Clear/New Order + charge
gating after Send (POS-19/D-04). This is the shared order-taking substrate for the phase.

Purpose: Today `pos-terminal.tsx` `ensureOrderId()` calls `useCreateOrder` on the first tap →
persists a DRAFT order that leaks into every list; every tap is a separate `addItem` (duplicate
lines); the table is mandatory (only floor-tile entry) and `type` is hardcoded DINE_IN; the terminal
never resets after send. This plan moves cart state into local React state and persists only on the
explicit Send/Charge action.

Output: a pure cart reducer (unit-tested), new order-type and table-selector components, a rewritten
terminal, order-panel quantity steppers + Clear/New Order + charge gating, and the PICKUP order type.
</objective>

| T-07.3-08 | Tampering | client cart prices/quantities | medium | mitigate | Cart is display-only; the server re-prices on addItem from the menu (unitPriceSnapshot server-set), so a tampered client price never persists. |
