## Decisions Made
- Closed filter scoped to `statuses=["CLOSED"]` only, not the full terminal set (`CLOSED`/`VOIDED`/`REFUNDED`) ‚Äî matches the chip's literal "Closed" label; voided/refunded orders remain reachable via their own `StatusBadge` elsewhere.
- Closed filter uses a second, `enabled`-gated `useOrderSummaries` query instance rather than re-pointing the always-on active-list query, so the fade-out invariant on the active list is never tripped by a filter-driven query-scope switch.
- Paid filter is a client-side `paymentStatus === "PAID"` filter over whichever query is currently displayed, per the plan's explicit instruction ‚Äî a paid-and-closed order is reached via the Closed filter instead.
- `table-select-combobox.tsx` gained an additive `availableOnly` prop (default `false`) rather than a new component; its one other caller (`order-panel.tsx`) is unaffected.
- The "closed/paid orders open read-only" requirement needed no new code ‚Äî it's already satisfied by 07.3-06/07.3-07's `OrderTableDetailDrawer`/`SettlementActions` `isSettled` gating, which Order Management's existing Open action already routes through.
