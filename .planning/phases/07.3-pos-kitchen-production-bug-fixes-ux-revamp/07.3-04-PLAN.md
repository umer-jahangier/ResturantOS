<objective>
Extend the pos-service Order-Management read model and add table assignment (POS-24 backend), retire
DRAFT from the summaries default (POS-16 backend), and emit the order's table number on the
send-to-KDS event (KDS-04 pos-side, additive field). All three edits share `OrderServiceImpl.java`,
so they are one plan to avoid a same-wave file conflict.

Purpose: `OrderSummaryDto` is deliberately thin (no settlement/payment/item data), the summaries
default filter still includes DRAFT, there is no assign-table capability anywhere, and the KDS ticket
has no table number because the event never carries it. This plan unblocks the Order-Management UI
(08) and the KDS table-number display (kitchen side lands in 05).

Output: an extended DTO with server-derived payment status + item quantity, a
`PATCH /orders/{id}/table` endpoint, a DRAFT-excluding default, and a `tableNumber` field on the
send-to-KDS payload.
</objective>
