    `FileSecurityConfig` (Security 7 lambda DSL): wire shared-lib `JwtAuthenticationFilter`; `/api/v1/files/**` requires authentication; permitAll actuator.

    `PlatformAdminClient` (`@FeignClient(name="platform-admin-service", url="${restaurantos.platform-admin-service.uri}", configuration=FeignSharedConfig.class)`): `GET /internal/platform/tenants/{tenantId}/status` → returns `{status, tier}` (includes storage_gb info indirectly — for v1, use a direct approach: GET the tenant's `storage_gb` limit). NOTE: platform-admin exposes `GET /internal/platform/tenants/{id}/status` which returns tier — derive limit from tier defaults. OR if platform-admin returns storage_gb directly, use that. For robustness, add a fallback: `getStorageLimitBytes(UUID tenantId)` that calls platform-admin and returns the limit in bytes. If platform-admin is unreachable, reject the upload (fail-closed).

    `QuotaService`: `checkQuota(UUID tenantId, long newFileSizeBytes)` — **atomic via Redis** (RESEARCH: prevent concurrent upload race):
      1. `limitBytes = platformAdminClient.getStorageLimitBytes(tenantId)`.
      2. `newTotal = redisTemplate.opsForValue().increment("storage:bytes:" + tenantId, newFileSizeBytes)`.
      3. If `newTotal > limitBytes` → `DECRBY` rollback, throw `QuotaExceededException("STORAGE_GB", newTotal - newFileSizeBytes, limitBytes)`.
      4. On upload failure after INCRBY, always DECRBY rollback. Cross-check `fileMetadataRepository.sumSizeBytesByTenantId()` periodically for drift correction (document in SUMMARY).

    `FileStorageService`:
      - `upload(MultipartFile file, UUID tenantId, UUID userId)`:
        1. Call `quotaService.checkQuota(tenantId, file.getSize())` — BEFORE touching MinIO.
        2. Generate `fileId = UUID.randomUUID()`.
        3. Sanitize filename (remove path separators, limit length).
        4. Compute `objectKey = tenantId + "/" + fileId + "/" + sanitizedFilename`.
        5. Compute SHA-256 hash of the file content.
        6. PUT to MinIO: `minioClient.putObject(PutObjectArgs.builder().bucket(bucket).object(objectKey).stream(inputStream, size, -1).contentType(contentType).build())`.
        7. Persist `FileMetadataEntity` (tenantId from TenantContext, uploadedBy=userId, objectKey, originalFilename, contentType, sizeBytes, sha256).
        8. Return `FileUploadResponse{fileId, objectKey, downloadUrl="/api/v1/files/{fileId}/download"}`.
      - `download(UUID fileId)`: lookup metadata (RLS-scoped), get object from MinIO by `objectKey`, return `InputStreamResource` with content-type.
      - `delete(UUID fileId)`: soft-delete metadata (set deleted_at); optionally remove from MinIO (or retain for compliance — mark as deleted).

    `FileDtos`: `FileUploadResponse{UUID fileId, String objectKey, String downloadUrl, long sizeBytes, String contentType}`, `FileListResponse`, `QuotaStatusResponse{long usedBytes, long limitBytes, double usedPercent}`.

    `FileController` (`@RestController @RequestMapping("/api/v1/files")`):
      - `POST /` (multipart) → `fileStorageService.upload(file, tenantId, userId)` → 201 with FileUploadResponse.
      - `GET /` → paginated list of file metadata for the tenant (RLS-scoped).
      - `GET /{id}/download` → stream file content from MinIO with correct Content-Type + Content-Disposition.
      - `DELETE /{id}` → soft-delete.
      - `GET /quota` → current usage vs limit for the tenant.
