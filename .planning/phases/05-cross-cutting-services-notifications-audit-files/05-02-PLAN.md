    - `010-create-audit-events.xml`: create table `audit_events` as PARTITIONED BY RANGE (occurred_at): `id BIGSERIAL`, `tenant_id UUID`, `branch_id UUID`, `user_id UUID`, `action VARCHAR NOT NULL` (event type), `resource_type VARCHAR`, `resource_id VARCHAR`, `before_state JSONB`, `after_state JSONB`, `ip_address VARCHAR`, `user_agent VARCHAR`, `occurred_at TIMESTAMPTZ NOT NULL DEFAULT NOW()`, `metadata JSONB`. Also: `GRANT INSERT ON audit_events TO audit_writer; REVOKE UPDATE, DELETE ON audit_events FROM audit_writer;` (the runtime user can ONLY insert).

      ```sql
      CREATE OR REPLACE FUNCTION prevent_audit_mutation() RETURNS trigger AS $$
      BEGIN RAISE EXCEPTION 'audit_events is append-only'; END;
      $$ LANGUAGE plpgsql;
      CREATE TRIGGER audit_events_immutable
        BEFORE UPDATE OR DELETE ON audit_events
        FOR EACH ROW EXECUTE FUNCTION prevent_audit_mutation();
      ```
      This enforces immutability even if someone bypasses the GRANT (defense in depth).
    - `012-audit-partitioning.xml`: a helper function `create_audit_partition(month DATE)` that creates a partition `audit_events_YYYY_MM` for a given month range. Create partitions for the current month through +12 months ahead. The archival job will call this to create future partitions and detach old ones.
    - `020-shared-infra-tables.xml`: `processed_events` (consumer dedup) per Doc 8 §8.9 — NON-RLS.

    `ProcessedEventService`: same idempotent pattern as notification — `boolean tryProcess(String consumerName, UUID eventId, Runnable action)`. Check `processedEventRepository.existsByConsumerNameAndEventId`; if exists skip; else run action + INSERT row, all within `@Transactional`.

    `AuditIngestionService`: `ingest(EventEnvelope<?> envelope)` → determine if `isAuditable(eventType)` (auditable types: `USER_LOGIN_SUCCEEDED`, `USER_LOGIN_FAILED`, `IMPERSONATION_STARTED`, `TENANT_PROVISIONED`, `RBAC_CHANGED`, `VOID_CREATED`, `REFUND_CREATED` + any event from `platform.topic` or `auth.topic`). Map envelope fields → `AuditEventEntity`: tenantId from envelope, branchId from payload, userId from payload/JWT claims embedded in payload, action=eventType, after_state=payload as JSONB, occurred_at from envelope timestamp. INSERT via repository (INSERT-only user — no save-and-flush with merge; use `persist` or `saveAndFlush` which does INSERT for new entities).

    `AllEventsConsumer` (`@RabbitListener(queues="audit.all-events.queue")`): parse `Message` → `EventEnvelope<Map<String,Object>>`. Call `processedEventService.tryProcess("audit.all-events", envelope.eventId(), () -> { tenantAwareMessageProcessor.process(envelope, env -> auditIngestionService.ingest(env)); })`. The queue is already bound to all 9 exchanges with `#` in rabbitmq-definitions.json — no binding changes needed.

    `AuditArchivalService` (`@Scheduled(cron="0 0 2 1 * ?")`): runs on the 1st of each month at 2am. 1) Create next month's partition (call `create_audit_partition`). 2) Detach partitions older than 7 years: query `pg_catalog.pg_inherits` / `pg_class` for partitions with suffix `_YYYY_MM` where date < now()-7years → `ALTER TABLE audit_events DETACH PARTITION audit_events_YYYY_MM`. Detached partitions remain in the archive schema for compliance but are no longer queryable by default. Log the detachment.

    `AuditInternalController` (`@RestController @RequestMapping("/internal/audit")`): `GET /events?tenantId=&from=&to=&page=&size=` → paginated audit events for compliance/admin query. This is optional for SC2 but useful for IT assertions and future admin UI.
