## Issues Encountered

- **Backend dev stack is stale** (running pre-07.1 code): gateway returns `503` from its FallbackController for `POST /pos/tills`, `/orders/{id}/close`, `/orders/{id}/void` (circuit-breaker tripping, most likely on stale pos-service erroring against the new request shapes), and `send-to-kds` never populates `items[].revisionNo`. These block S2/S5/S6 and S4's exact revision text. **Not a frontend defect** — deferred to the consolidated phase-end backend rebuild + re-run of `e2e/pos-settlement.spec.ts`.
- **Duplicate `@RestControllerAdvice` in pos-service** (discovered): `PosGlobalExceptionHandler` (RFC7807 ProblemDetail, code `FORBIDDEN`) and shared-lib `GlobalExceptionHandler` (code `PERMISSION_DENIED`) both handle the same exceptions with different shapes — ambiguous runtime resolution. Frontend branches on HTTP status to sidestep it; flagged for a future backend plan (no Java touched here).
