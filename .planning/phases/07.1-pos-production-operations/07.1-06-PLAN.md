<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cashier UI → pos-service | charge/void/refund actions gated by PermissionGuard (UI) + OPA (server) |
| offline outbox → IndexedDB | queued mutations replayed on reconnect |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-07.1f-01 | Elevation of Privilege | settlement-actions (charge/void/refund) | high | mitigate | PermissionGuard hides actions the user lacks (UI), but authority stays server-side (OPA fail-closed via pos.rego); UI guard is defence-in-depth, never the sole gate |
| T-07.1f-02 | Repudiation | offline queued settlement actions | medium | mitigate | Queued mutations carry their idempotency keys (clientOrderId / clientFireId); server dedups replays — no duplicate charge/fire on reconnect |
| T-07.1f-03 | Information Disclosure | void-403 error surfacing | low | accept | Inline "no permission" banner reveals only the caller's own authorization state; no cross-tenant data; stale-JWT revoke lag is the existing accepted 15-min TTL tradeoff (RESEARCH.md Security Domain) |
| T-07.1f-SC | Tampering | npm installs | high | mitigate | No new packages (RESEARCH.md audit: none) |
</threat_model>
