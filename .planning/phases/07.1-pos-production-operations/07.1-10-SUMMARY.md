# Phase 07.1 Plan 10: Table Floor View — 3-State Tile Lifecycle + Dual Tap Paths Summary

**Migrated `table-floor-view.tsx` off hardcoded Tailwind palette classes onto semantic tokens, added the three-state tile lifecycle (AVAILABLE/OCCUPIED-with-derived-status-badge/NEEDS_BUSSING) each with a lucide icon, and completed both POS-10 entry paths: an AVAILABLE tap binds the table and starts a new order on the Terminal tab, while an OCCUPIED/NEEDS_BUSSING tap opens the exact same shared `OrderTableDetailDrawer` built in plan 09 — never a second table-detail UI.**

- **Completed:** 2026-07-11

## Accomplishments

- **Semantic-token migration (UI-SPEC §2 Color contract):** replaced every hardcoded `green-400/50/100`, `orange-400/50/100`, `blue-100/700` class with `border-success bg-success/10`, `border-info bg-info/10`, `border-warning bg-warning/10` — matching the phase's Color System exactly.
- **Three-state tile lifecycle with icon-per-state:** `AVAILABLE` → `CheckCircle2` + "Available"; `OCCUPIED` → `Utensils` + "Occupied" **plus** a derived-order-status `StatusBadge` (fetched via `useTableDetail(tableId)`, enabled-gated to OCCUPIED tiles only); `NEEDS_BUSSING` → `Sparkles` + "Needs Bussing". No tile relies on color alone (WCAG); every tile keeps the pre-existing `.touch-target` 44px floor.
- **AVAILABLE tap path:** calls the existing `onTableSelect` prop unchanged — `page.tsx` (plan 06) binds `selectedTableId` and switches to the Terminal tab exactly as it already did; this plan made no changes to that contract.
- **OCCUPIED/NEEDS_BUSSING tap path (new):** `TableFloorView` now owns a local `detailTable` state and renders the shared `OrderTableDetailDrawer` (plan 09) directly, passing `tableId`/`tableName` — resolving the table's active order via the same component Order Management uses, with zero duplicated detail-rendering logic.
- **New test file** (`table-floor-view.test.tsx`, 4 tests): all three tile states render with correct semantic classes + icons + the OCCUPIED derived-status badge; an AVAILABLE tap calls `onTableSelect` with the tapped table; an OCCUPIED tap opens the shared drawer and does NOT call `onTableSelect`; the empty state still renders when no tables exist.
