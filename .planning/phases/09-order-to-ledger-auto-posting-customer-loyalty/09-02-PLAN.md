  <action>
    `LoyaltyService`:
    - `accrueForOrder(customerId, orderId, totalPaisa)` — compute points from tier config; insert ACCRUAL transaction; update points_balance + lifetime_spend; call `checkTierUpgrade()`
    - `debitForRefund(customerId, orderId, refundPaisa)` — proportional point debit; insert DEBIT transaction
    - `checkTierUpgrade(account)` — compare lifetime_spend to tier_config thresholds; upgrade BRONZE→SILVER→GOLD

    `OrderClosedLoyaltyConsumer` on `crm.order-closed.queue` (exists): skip if payload.customerId null; idempotent via ProcessedEventService.

    Add `crm.order-refunded.queue` bound to `pos.topic` / `pos.order.refunded` in definitions.json.
    `OrderRefundedLoyaltyConsumer`: debit loyalty on refund.

    `PromotionEngine.evaluate(EvaluatePromotionRequest)`:
    - Filter active promotions by: now between start/end, day-of-week, hour window, tier_filter (if customer tier set), menu_item_ids intersection with order items
    - Apply best eligible discount (PERCENT or FIXED cap at subtotal)
    - Return `{ discountPaisa, appliedPromotionIds[] }`
    Wire into `CrmInternalController POST /internal/crm/promotions/evaluate`.

    `PromotionController` (`/api/v1/crm/promotions`): CRUD for promotions (crm.promotion.manage permission).

    `FeedbackService` + `FeedbackController` (`/api/v1/crm/feedback`):
    - POST submit feedback (customer_id, order_id, rating, comment) — callable from POS post-payment UI
    - GET paginated list for CRM reporting (crm.customer.view)

    `CrmLoyaltyIT` (Testcontainers Postgres + RabbitMQ):
    1. Create customer + loyalty account
    2. Publish ORDER_CLOSED with customerId + totalPaisa → points accrued, lifetime_spend updated
    3. Publish ORDER_CLOSED again same eventId → dedup, no double accrual
    4. Accrue enough to cross Silver threshold → tier upgraded
    5. Publish ORDER_REFUNDED → points debited

    `PromotionEngineIT`:
    - Create tier-specific + time-window promotion → evaluate returns expected discountPaisa
    - Promotion outside time window → zero discount

    **POS contract note for Phase 7:** Document in SUMMARY that POS must set `customerId` on ORDER_CLOSED payload and call `/internal/crm/promotions/evaluate` before finalizing payment.
