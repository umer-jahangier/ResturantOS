<objective>
Make the "silent no-op" gap confirmed in 08-CONTEXT.md observable: add `GET /api/v1/inventory/recipes/coverage` so an operator can see which active menu items lack an effective recipe (INV-11), and change `DepletionService.deplete` so an ORDER_CLOSED line with no effective recipe still lets covered lines deplete normally while ALWAYS publishing `DEPLETION_INCOMPLETE` — replacing the current behavior where an order with zero resolvable recipes returns with no signal at all (D-03).

Purpose: 08-05 built accurate depletion but made missing-recipe coverage invisible; this plan is what makes a coverage gap discoverable and alertable instead of a silent, permanent stock-count drift.
Output: `RecipeService.getCoverage()` + its endpoint; `DepletionService.deplete`'s DEPLETION_INCOMPLETE branch; `InventoryEventPayloads.DepletionIncompletePayload`.
</objective>
