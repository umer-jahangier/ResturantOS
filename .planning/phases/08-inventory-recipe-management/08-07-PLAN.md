<objective>
Deliver inter-branch stock transfers (INV-05): ship (decrement source, mark in-transit), receive (increment destination, recompute destination MAC), and variance handling for shortfalls between shipped and received quantities — each step publishing the corresponding `TRANSFER_SHIPPED` / `TRANSFER_RECEIVED` / `TRANSFER_VARIANCE` event carrying the Inventory-in-Transit (`1320`) valuation so Phase 9's finance consumer can post the GL entries.

Purpose: Multi-branch restaurants move stock between locations; the inventory-side movement + in-transit state + variance detection must be accurate and event-published, decoupled from the GL posting (Phase 9).
Output: Transfer model + ship/receive/variance service + controller, with an end-to-end lifecycle IT.
</objective>
