<objective>
Deliver versioned recipes/BOM (INV-02): the Recipe + RecipeLine model, CRUD to create recipe versions, and the D-01 effective-version resolution query (`effective_from <= atInstant ORDER BY effective_from DESC`) that depletion (08-05) uses with the order's `closedAt`.

Purpose: Depletion must select the recipe version effective at order time, not the current one, so a mid-service recipe edit does not retroactively change how already-placed orders deplete.
Output: Recipe model + CRUD + the effective-version resolution seam, with unit and integration coverage.
</objective>
