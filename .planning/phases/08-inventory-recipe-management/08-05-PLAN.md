<objective>
Build the `ORDER_CLOSED` depletion consumer (INV-03): an idempotent RabbitMQ consumer that, per closed order, resolves the effective recipe version at `closedAt` (D-01), converts recipe lines to base quantity, locks stock rows in a deadlock-safe order, walks lots FEFO (flooring each lot at zero while the aggregate may go negative, D-02), values COGS at the aggregate MAC (D-04/Pitfall 9), writes DEPLETION movements, queues LOW_STOCK_ALERT, and publishes `STOCK_DEPLETED` through the transactional outbox — all exactly once per event.

Purpose: This is the core value loop — POS sale → inventory depletion with accurate quantities and moving-average COGS — and the event Phase 9 will consume to auto-post revenue+COGS.
Output: The consumer, the depletion domain service (lock ordering + FEFO + MAC-COGS + idempotency), and the UOM conversion helper, with unit + integration proof.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RabbitMQ → OrderClosedConsumer | Untrusted at-least-once ORDER_CLOSED delivery crosses into stock mutation |
| concurrent consumers → stock row | Two order closes can race on the same ingredient's qty + MAC |
| DB connection → tenant rows | Depletion must run under the event's tenant/branch context |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-8-IDEM | Repudiation / Tampering | OrderClosedConsumer + processed_events | high | mitigate | tryProcess(CONSUMER_NAME='inventory.depletion', eventId) dedup — duplicate delivery depletes once; proven by DepletionConsumerIT duplicate-publish (must_haves.truths #5). |
| T-8-RACE | Tampering (data integrity) | IngredientBranchStock qty + avg_cost | high | mitigate | PESSIMISTIC_WRITE findForUpdate holds the row for the whole read-modify-write; distinct ingredientIds locked in sorted order to avoid deadlock (must_haves.truths #2, prohibitions #2). |
| T-8-XTEN | Elevation of Privilege | stock/lot/movement mutation | high | mitigate | TenantAwareMessageProcessor sets the RLS GUC + tenant filter in the listener transaction; repositories scope tenantId; FORCE RLS on all tables. |
| T-8-SC | Tampering (supply chain) | dependencies | low | accept | No new packages. |
</threat_model>
